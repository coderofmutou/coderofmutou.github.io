# 知识库图片治理流水线（Image Pipeline）

## 适用场景

- 文档中存在外链图片，需要本地化以避免失效
- 仓库图片体积大，构建慢，需要统一压缩为 `.webp`
- 图片路径混乱，出现 404 或部署后不显示
- 需要识别并清理无引用图片，减少仓库膨胀

## 输入要求

- 处理范围（单文件 / 单目录 / 全仓库）
- 优先目标（稳定性优先 / 体积优先 / 画质优先）
- 是否允许删除无引用图片（默认先报告后删除）

## 标准流程

1. **盘点现状**
   - 扫描目标范围中的图片引用类型（外链、相对路径、绝对路径）。
   - 识别高风险路径（中文路径、空格、大小写不一致）。

2. **外链图片本地化**
   - 将外链下载到文档同级 `.assets/`。
   - 将 Markdown 引用替换为相对路径。

3. **格式统一与压缩**
   - 优先转为 `.webp`，保持清晰可读。
   - 对超大图片执行压缩，兼顾体积与可读性。

4. **引用修复**
   - 校验每个图片引用是否可达。
   - 修复目录移动导致的路径断裂。

5. **无引用图片治理**
   - 先生成“未使用图片清单”。
   - 经确认后再删除，避免误删历史素材。

6. **回归检查**
   - 抽查关键页面渲染结果。
   - 确认构建产物中图片可正常加载。

## 建议工具映射（本仓库）

可结合以下脚本完成流水线步骤：

- `utils/compress/download_external_images.py`
- `utils/compress/compress_image2webp.py`
- `utils/compress/search_unused_images.py`
- `utils/compress/integrated_process.py`

> 若处理目标是 Java 学习章节，优先从单目录试跑，再扩展到全仓库。

## 决策分支

- **清晰度明显下降**：保留原图并仅替换通过质检的 webp。
- **同名文件冲突**：按文档名或哈希重命名，避免覆盖。
- **历史文章 permalink 已发布**：不改 permalink，仅修图与引用。
- **删除风险高**：先输出报告，不自动删除。

## 完成标准（Quality Gate）

- [ ] 目标范围内外链图片已本地化（如要求）
- [ ] 新图片路径均为相对路径且可访问
- [ ] `.webp` 转换后关键图可读性达标
- [ ] 未使用图片清单已产出并人工确认
- [ ] 修复后页面无明显图片加载失败

## 产出物

- 已更新的 Markdown 图片引用
- 本地化后的 `.assets/` 图片集
- 未使用图片报告（可选附删除建议）
- 简短处理摘要（处理范围、转换数量、风险项）

## 示例调用

- `/knowledge-image-pipeline 修复这篇笔记的图片 404，仅做路径修复不做删除`
