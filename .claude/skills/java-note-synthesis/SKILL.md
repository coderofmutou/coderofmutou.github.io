# Java 学习笔记整合（Synthesis）

## 适用场景

- 多份课程讲义、博客、官方文档需要合并为一篇个人笔记
- 需要处理版本差异（如 Spring Boot 2.x vs 3.x）
- 现有文档内容冗余、顺序混乱、可读性差
- 希望统一为可检索、可维护、可扩展的知识资产

## 输入要求

- 目标主题与边界（如「只覆盖 Spring Security 6.x 核心配置」）
- 原始资料列表（链接、文本、旧笔记文件）
- 目标落盘路径（如 `docs/03.微服务生态/10.安全框架/`）
- 是否保留历史版本对照（默认保留）

## 标准流程

1. **归档原始资料**
   - 先确认资料范围，排除与主题无关内容。
   - 识别目标版本（JDK、Spring Boot、框架主版本）。

2. **抽取知识骨架**
   - 按「是什么 → 为什么 → 怎么做 → 常见坑」建立主线。
   - 先列目录，再填内容，避免“想到哪写到哪”。

3. **去重与去噪**
   - 同义概念只保留一处主定义。
   - 删除广告、讲师口播、无关截图、重复段落。

4. **代码示例归一化**
   - 优先保留最小可运行示例。
   - 代码块必须标语言（`java`、`xml`、`yaml`、`bash`）。
   - 对关键配置补“为什么这样配”的一句解释。

5. **版本差异处理**
   - 小差异：在同节中用引用块标注。
   - 大差异：分节或分文件表达，并标明版本适用范围。

6. **写入文档规范层**
   - frontmatter 必填项完整（title/date/permalink/categories/tags/author）。
   - 标题层级清晰，全文仅一个 H1。
   - 图片路径使用相对引用，资源放同级 `.assets/`。

7. **收尾与交付**
   - 校对目录、链接、代码块语言、版本注释。
   - 必要时同步目录页和导航配置。

## 决策分支

- **资料结论冲突**：官方文档优先，其次最新版实践；旧方案保留为“历史说明”。
- **篇幅过长（>500 行）**：按主题拆分为多文件，并在主文档加导航。
- **示例代码重复**：保留最短可运行版本，其余转为“扩展阅读”。
- **主题跨层级**：核心正文聚焦单主题，跨主题内容链接到其他笔记。

## 完成标准（Quality Gate）

- [ ] 文档结构能在 1 分钟内看懂主线
- [ ] 关键概念无重复定义
- [ ] 每段代码都带语言标识且上下文完整
- [ ] 版本边界清晰（如 5.x/6.x）
- [ ] frontmatter、permalink、分类标签完整且可用
- [ ] 无失效链接、无明显噪声文本

## 产出物

- 一篇可发布的目标笔记（或一组拆分后的系列笔记）
- 版本差异说明块（如有）
- 必要的后续维护清单（待补章节、待验证示例）

## 示例调用

- `/java-note-synthesis 整合三份 MyBatis 动态 SQL 资料，输出一篇面向面试与实战的笔记`
- `/java-note-synthesis 把 Spring Security 5.x 旧笔记升级到 6.x，保留迁移差异`
