# 中英文混排排版规范

知识库笔记的排版硬规则。分两部分：**textlint 能自动修的**（跑命令即可）和 **textlint 修不到的盲区**（写作时手动保证）。

---

## 一、textlint 能自动修复（跑命令）

已配置两条规则，覆盖大部分中英文混排间距：

| 规则 | 作用 | 示例 |
|------|------|------|
| `ja-space-between-half-and-full-width` | 中文（全角）与英文/数字（半角）间补空格 | `这是AI` → `这是 AI` |
| `ja-space-around-code` | 行内代码前后补空格 | `使用`npm`命令` → `使用 `npm` 命令` |

```bash
cd utils/textlint
npm run kb:lint:one -- ../../docs/04.实用工具/10.Git.md    # 单篇检查
npm run kb:lint:one:fix -- ../../docs/04.实用工具/10.Git.md # 单篇修复（覆盖原文件）
npm run kb:lint:fix                                        # 全仓库修复
```

> ⚠️ `kb:lint:fix` 直接覆盖原文件，跑前确保 `git status` 干净。

## 二、textlint 修不到的盲区（手动保证）

这些规则跨 AST 节点边界或需语境判断，textlint 无法自动处理，写作时手动遵守。

### 1. `**加粗**` / `*斜体*` 标记边界的中英文间距

textlint 无法检测 Markdown 标记符号与中文之间的间距。

```markdown
# ❌ 错误
这是**Java**相关的技术

# ✅ 正确（标记两侧与中文间留空格）
这是 **Java** 相关的技术
```

**原则**：`**` / `*` / `` ` `` 等标记符号与相邻中文之间，按中英文间距规则留空格。

### 2. 全角/半角标点选择

textlint 不强制标点全半角，需按语境手动选择：

- **中文语境**：用全角标点（`，。：；？！（）""''`）
- **英文语境/代码/命令/变量名**：用半角标点（`,.:;?!()""''`）
- **代码块内**：一律不约束，保持代码原样

```markdown
# ✅ 中文语境全角
Spring Boot 是一个框架，用于快速构建应用。

# ✅ 代码/命令半角
运行 `npm install` 安装依赖。

# ❌ 混用
Spring Boot是一个框架,用于快速构建应用.
```

### 3. 代码块内不适用任何排版规则

围栏代码块（``` / ~~~）内的中英文间距、全半角标点**不受任何规则约束**，保持代码原样。`md:fix` 和 textlint 都会跳过代码块内容。

### 4. 行内代码的使用边界

仅在涉及**代码、命令、变量、配置项、文件名**时用反引号；普通英文/数字正常混用即可。

```markdown
# ✅ 用反引号
数据库的主键 `id` 必须唯一
配置项 `spring.datasource.url`

# ❌ 不该用反引号
我们使用 `Python 3.12` 来实现并发   # Python 3.12 是普通英文+数字，不需反引号
```

## 三、通用 Markdown 结构规则

这些 `md:check` / `md:fix` 能部分自动处理，但写作时就应遵守：

| 规则 | 要求 | 工具 |
|------|------|------|
| 标题层级 | H1 仅用于文档标题，正文从 H2 开始 | `md:check` 报多 H1 |
| 标题空行 | 标题上下各保留一行空行 | `md:fix` 自动补 |
| 列表空格 | 列表符号与内容间加空格（`- 项目` 非 `-项目`） | `md:fix` 自动补 |
| 连续空行 | 不超过 2 个连续空行 | `md:fix` 自动压缩 |
| 代码块语言 | 所有代码块必须带语言标注（```java / ```xml / ```yaml） | 人工保证 |
| 行宽 | 单行不超过 80–100 字符，必要时手动换行 | 人工保证 |
| 段落间距 | 段落间空一行 | 人工保证 |
| 图片路径 | 用相对路径，避免绝对路径 | `md:check` / `kb:scan` 报告 |

---

## 四、写作自检清单

发布前逐项确认：

- [ ] 中英文之间有空格（`kb:lint:fix` 可自动修）
- [ ] 行内代码前后有空格（`kb:lint:fix` 可自动修）
- [ ] `**加粗**` / `*斜体*` 两侧与中文间有空格（手动）
- [ ] 中文语境用全角标点，代码/命令用半角（手动）
- [ ] 代码块均带语言标识（手动）
- [ ] 代码块内排版未受规则破坏（工具自动豁免）
- [ ] 行内代码仅用于代码/命令/变量/配置项（手动）
