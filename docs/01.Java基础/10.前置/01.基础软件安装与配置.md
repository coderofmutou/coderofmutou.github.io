# 基础软件安装与配置

本指南涵盖 Java 开发环境的完整搭建过程，包括 JDK、Maven、IDEA、MySQL、Tomcat、Git 和 Docker 的安装与配置。完成本文配置后，即可运行本地 Java 项目、管理依赖、连接数据库，并使用容器化中间件。

## 环境变量配置通用思路

Windows 下大部分 Java 开发工具都遵循同一套环境变量配置模式：

1. **添加 `XXX_HOME`**：指向软件安装目录，便于统一管理和版本切换。
2. **追加 `Path`**：将可执行文件目录加入 `Path`，使命令行能全局调用。
3. **验证**：重启终端后执行 `xxx -version` 或 `xxx -v` 确认生效。

::: tip 为什么需要 JAVA_HOME？
许多工具（如 Maven、Tomcat）和启动脚本会读取 `JAVA_HOME` 来定位 JDK。直接写 `Path` 虽然也能调用 `java` 命令，但无法被其他工具复用。
:::

## JDK 安装与配置

### 下载与安装

1. 访问以下任一站点下载 JDK：
   - [Oracle 官网](https://www.oracle.com/java/technologies/downloads/)
   - [OpenJDK](https://jdk.java.net/)
   - [国内镜像（inJDK）](https://injdk.cn)
2. 推荐安装 **JDK 17** 或 **JDK 21**（均为长期支持版本，LTS）。
3. 安装完成后配置环境变量。

::: tip JDK 版本选择建议
- **JDK 8**：生态最成熟，大量老项目仍在使用，但已停止官方免费长期支持，适合维护旧项目。
- **JDK 11**：LTS 版本，模块化和垃圾回收器改进明显，适合从 JDK 8 升级的团队。
- **JDK 17**：当前主流 LTS，Spring Boot 3.x 的最低要求，推荐作为新项目的默认选择。
- **JDK 21**：最新 LTS，引入虚拟线程（Virtual Threads）等特性，适合对性能与并发有较高要求的场景。
:::

### 配置环境变量

#### 1. 配置 JAVA_HOME

在系统环境变量中添加：

```text
变量名：JAVA_HOME
变量值：C:\Program Files\Java\jdk-17  # 根据实际安装路径调整
```

#### 2. 配置 Path

在系统变量 `Path` 中添加：

```text
%JAVA_HOME%\bin
```

#### 3. 验证安装

打开命令提示符，执行：

```bash
java -version
javac -version
```

## Maven 安装与配置

### 下载与安装

1. 访问 [Maven 官网](https://maven.apache.org/download.cgi) 下载二进制包。
2. 解压到指定目录，如：`C:\Softwares\DevTools\apache-maven-3.9.6`。

### 配置环境变量

#### 1. 配置 MAVEN_HOME

```text
变量名：MAVEN_HOME
变量值：C:\Softwares\DevTools\apache-maven-3.9.6
```

#### 2. 配置 Path

在系统变量 `Path` 中添加：

```text
%MAVEN_HOME%\bin
```

#### 3. 验证安装

```bash
mvn -v
```

### 配置本地仓库与镜像

编辑 `conf/settings.xml` 文件。一个适合本地开发的完整配置示例如下：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<settings xmlns="http://maven.apache.org/SETTINGS/1.2.0"
          xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
          xsi:schemaLocation="http://maven.apache.org/SETTINGS/1.2.0 https://maven.apache.org/xsd/settings-1.2.0.xsd">

  <!-- 本地仓库路径，默认在 ~/.m2/repository -->
  <localRepository>C:/Softwares/DevTools/MavenLocalRepository</localRepository>

  <mirrors>
    <mirror>
      <id>aliyunmaven</id>
      <mirrorOf>*</mirrorOf>
      <name>阿里云公共仓库</name>
      <url>https://maven.aliyun.com/repository/public</url>
    </mirror>
  </mirrors>

</settings>
```

::: tip Maven 仓库类型
- **本地仓库**：本机存放依赖的目录，默认在 `~/.m2/repository`。
- **中央仓库**：Maven 官方维护的公共仓库，坐标通过 `groupId:artifactId:version` 定位。
- **镜像仓库**：国内访问中央仓库较慢时，可用阿里云镜像等代理加速下载。
- **私服仓库**：企业内网部署的 Nexus / Artifactory，用于管理私有依赖和统一镜像。
:::

::: note 💡 个人注
`<mirrorOf>*</mirrorOf>` 表示所有仓库请求都走阿里云镜像。如果公司内网有私服，可将 `<mirrorOf>` 改为 `central`，避免镜像私服导致私有依赖无法下载。
:::

## IntelliJ IDEA 配置

### JVM 参数调优

编辑 `idea64.exe.vmoptions` 文件（位于 IDEA 安装目录的 `bin` 目录下）：

```text
-Xms1024m
-Xmx4096m
-XX:ReservedCodeCacheSize=512m
-XX:+UseG1GC
```

### 常用设置

进入 **Settings**（`Ctrl+Alt+S`）依次配置：

- **鼠标滚轮调字体**：`Editor > General`，勾选 *Change font size with Ctrl+Mouse Wheel*。
- **自动导包**：`Editor > General > Auto Import`，勾选 *Add unambiguous imports on the fly*。
- **多行显示 Tab**：`Editor > General > Editor Tabs`，将 *Show tabs in* 设为 *Multiple Rows*。
- **最大 Tab 数**：`Editor > General > Editor Tabs`，修改 *Tab limit* 的值（默认 10，可按需调大）。
- **文件编码**：`Editor > File Encodings`，将 *Global Encoding*、*Project Encoding* 以及 *Properties Files* 的编码均设为 UTF-8；同时勾选 *Transparent native-to-ascii conversion*，使 `.properties` 文件可直接编写中文（IDEA 自动转存为 `\uXXXX` 格式，避免乱码）。

### Maven 集成配置

进入 `File → Settings`（`Ctrl+Alt+S`） → `Build, Execution, Deployment → Build Tools → Maven`：

- **Maven home path**：改为本地安装目录，如 `C:\Softwares\DevTools\apache-maven-3.9.6`。
- **User settings file**：选择对应的 `conf/settings.xml`（含阿里云镜像与本地仓库路径配置）。
- **Local repository**：确认展示的路径与 `settings.xml` 中的 `<localRepository>` 一致。

### 推荐插件

| 插件名称 | 作用 |
|---------|------|
| Lombok | 通过注解生成 getter / setter / toString 等样板代码。 |
| Maven Helper | 可视化解决 Maven 依赖冲突。 |
| Rainbow Brackets | 彩虹括号，便于阅读嵌套结构。 |
| Key Promoter X | 提示可替代鼠标的快捷键，帮助养成键盘操作习惯。 |
| .ignore | 便捷生成 `.gitignore` 文件。 |
| Translation | 选中单词即可翻译，阅读英文文档时提高效率。 |

::: tip 插件安装方式
进入 **File → Settings → Plugins**（`Ctrl+Alt+S`），在 Marketplace 中搜索插件名称，点击 **Install** 后重启 IDEA 即可生效。
:::

## MySQL 安装与配置

### 下载与安装

以 MySQL 8.0 为例：

1. 访问 [MySQL 官网](https://dev.mysql.com/downloads/mysql/) 下载。
2. 运行安装包，选择安装类型时选 **Custom**（自定义）。
3. 在产品选择界面勾选 **MySQL Server 8.0**，可额外勾选 MySQL Workbench（图形化管理工具）。
4. 下载并安装完成后进入配置向导：
   - **Config Type**：选 `Development Computer`（内存占用适中，适合开发环境）。
   - **Port**：保持默认 `3306`。
   - **Authentication Method**：选 `Use Strong Password Encryption`（MySQL 8.0 默认，安全性更高）。
   - **Root 密码**：设置并记录 root 账户密码。
   - **Windows Service**：勾选 `Start the MySQL Server at System Startup`（开机自启）。
5. 点击 **Execute** 完成配置。

### 环境变量配置

在系统变量 `Path` 中添加：

```text
C:\Softwares\DevTools\MySQL\MySQL Server 8.0\bin
```

### 验证安装

```bash
mysql -u root -p
```

## Tomcat 安装与配置

::: tip 适用范围说明
Tomcat 独立部署主要用于 **传统 JavaWeb 开发**（如 Servlet、JSP 阶段）。一旦进入 Spring Boot 开发阶段，项目内置了 Tomcat，**无需安装独立 Tomcat**。
:::

### 下载与安装

1. 访问 [Apache Tomcat 官网](https://tomcat.apache.org/) 下载。
2. 解压到指定目录，如：`C:\Softwares\DevTools\apache-tomcat-9.0.85`。

### 配置环境变量

#### 1. 配置 CATALINA_HOME

```text
变量名：CATALINA_HOME
变量值：C:\Softwares\DevTools\apache-tomcat-9.0.85
```

#### 2. 配置 Path

在系统变量 `Path` 中添加：

```text
%CATALINA_HOME%\bin
```

### 启动与验证

执行以下命令启动 Tomcat：

```bash
startup.bat
```

启动后访问 `http://localhost:8080` 验证是否成功。

### 中文编码问题处理

如果遇到中文乱码问题，修改 `conf/logging.properties` 文件：

```properties
! 将 UTF-8 替换为 GBK
java.util.logging.ConsoleHandler.encoding = GBK
```

::: note Tomcat 与 Spring Boot 内置容器的关系
Spring Boot 默认将 Tomcat 作为内嵌 Servlet 容器打包进应用。内嵌 Tomcat 的版本由 Spring Boot 的 Starter 管理，通常无需手动安装和配置独立 Tomcat。
:::

## Git 安装与配置

Git 是开发必备工具，用于代码版本管理与团队协作。

### 下载与安装

1. 访问 [Git 官网](https://git-scm.com/downloads) 下载对应系统安装包。
2. Windows 安装时推荐勾选 **Git Bash Here**，便于在任意目录打开 Git 命令行。
3. 安装过程中编辑器可选常用的 VS Code 或 IDEA。

### 基本配置

安装完成后，打开 Git Bash，配置全局用户信息：

```bash
git config --global user.name "你的用户名"
git config --global user.email "你的邮箱@example.com"
```

### 配置 SSH 密钥（可选）

访问 GitHub / Gitee 等远程仓库时，使用 SSH 协议可避免每次输入密码：

```bash
# 生成密钥（默认回车即可）
ssh-keygen -t ed25519 -C "你的邮箱@example.com"

# 查看公钥内容，复制后粘贴到 GitHub/Gitee 设置中
cat ~/.ssh/id_ed25519.pub
```

### 验证安装

```bash
git --version
```

### 配置代理（可选）

在公司内网或访问 GitHub 较慢时，可配置 Git 代理：

```bash
# 设置 HTTP 代理
git config --global http.proxy http://127.0.0.1:7890
# 设置 HTTPS 代理
git config --global https.proxy http://127.0.0.1:7890
# 取消 HTTP 代理
git config --global --unset http.proxy
# 取消 HTTPS 代理
git config --global --unset https.proxy
```

::: warning 注意
代理端口号需与本地代理工具实际端口一致，上述 `7890` 仅为示例。
:::

> 详细的 Git 命令使用（分支管理、远程协作等）请参考 [Git 常用命令](/tools/git/)。

## Docker 安装与配置

Docker 是容器化平台，进入微服务阶段后，常需用其快速启动 Nacos、Redis、MySQL 等中间件。

### 安装 Docker Desktop（Windows）

1. 访问 [Docker 官网](https://www.docker.com/products/docker-desktop/) 下载 Docker Desktop。
2. 安装前需开启 **WSL 2**（Windows 10/11 推荐）或 **Hyper-V** 虚拟化支持。
3. 安装完成后，在 Docker Desktop 设置中配置镜像加速。

### 配置镜像加速

在 Docker Desktop → Settings → Docker Engine 中添加：

```json
{
  "registry-mirrors": [
    "https://mirror.ccs.tencentyun.com",
    "https://hub-mirror.c.163.com"
  ]
}
```

::: warning 镜像源可用性提示
Docker Hub 国内镜像源地址会随网络政策变化，若上述地址无法使用，请搜索当前可用的镜像加速服务。
:::

### 验证安装

```bash
docker version
docker run hello-world
```

### 常用命令速查

```bash
# 查看本地镜像
docker images
# 查看运行中的容器
docker ps
# 启动 Redis 容器
docker run -d --name redis -p 6379:6379 redis:latest
# 停止容器
docker stop redis
# 删除容器
docker rm redis
```

### 使用 Docker Compose 启动中间件（可选）

进入微服务阶段后，常用 `docker-compose` 同时启动 Redis、MySQL、Nacos 等中间件。以下是一个最小示例：

```yaml
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    container_name: mysql-dev
    environment:
      MYSQL_ROOT_PASSWORD: root
    ports:
      - "3306:3306"
  redis:
    image: redis:latest
    container_name: redis-dev
    ports:
      - "6379:6379"
```

启动命令：

```bash
docker-compose up -d
```

::: note
Docker 命令详情可参考 [Docker 官方文档](https://docs.docker.com/)。
:::

完成所有配置后，请验证以下命令是否正常执行：

```bash
java -version
javac -version
mvn -version
git --version

docker version
```

:::

## 常见问题

### Q: JDK 安装后 java 命令无效？

A: 检查 `JAVA_HOME` 和 `Path` 配置是否正确，**重启命令提示符**后再次验证。部分旧版 Windows 还需注销或重启系统。

### Q: Maven 下载依赖失败？

A: 检查网络连接和镜像源配置，尝试清理本地仓库缓存（删除 `.m2/repository` 下报错的目录后重试）。

### Q: IDEA 启动缓慢？

A: 调整 VM 参数，增加内存分配，关闭不必要的插件。机械硬盘用户建议将 Maven 本地仓库和 IDEA 缓存目录迁移到 SSD。

### Q: MySQL 无法连接？

A: 检查服务是否启动，端口是否被占用，防火墙设置。若使用 MySQL 8.0 且连接工具较旧，可能需要更新驱动或修改认证插件。

### Q: Tomcat 中文乱码？

A: 修改 `logging.properties` 文件编码设置，检查应用编码配置。Spring Boot 项目无需独立 Tomcat，内嵌容器编码由 `server.servlet.encoding.charset=UTF-8` 控制。

### Q: Git 提交时提示 "Please tell me who you are."？

A: 尚未配置全局用户信息，执行 `git config --global user.name "xxx"` 和 `git config --global user.email "xxx"` 即可。

### Q: Docker Desktop 无法启动（WSL 2 报错）？

A: 以管理员身份运行 PowerShell，执行 `wsl --install` 安装 WSL 2，重启后再启动 Docker Desktop。若 CPU 不支持虚拟化，还需在 BIOS 中开启 VT-x/AMD-V。
